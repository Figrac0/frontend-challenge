Проект представляет собой интерфейс для просмотра изображений котиков с двумя основными сценариями:

- просмотр общей ленты котиков
- сохранение понравившихся изображений в избранное с хранением на клиенте

## Что сделано по условию задания

Ниже прямая сверка с исходным ТЗ.

### Обязательная часть

- По умолчанию открывается вкладка `Все котики`.
- У каждой карточки есть возможность добавить котика в избранное и убрать его из избранного.
- Избранные котики хранятся на клиенте в `localStorage`.
- На вкладке `Любимые котики` отображаются все сохраненные изображения.
- Интерфейс построен на `React + TypeScript + Vite`.

### Плюсы из задания, которые тоже реализованы

- Адаптивная верстка для десктопа, планшета и мобильных экранов.
- Бесконечная прокрутка в общей ленте.

## Что дополнительно реализовано

Кроме базового ТЗ, в проект добавлены функции, которые улучшают UX и делают решение сильнее с продуктовой точки зрения.

### Интерфейс и UX

- Липкий header с плавным поведением при скролле.
- Анимированное переключение вкладок.
- Анимированное появление карточек.
- Живая анимация кнопки лайка с «кошачьим» эффектом.
- Аккуратный анимированный фон без перегрузки интерфейса.
- Кастомный favicon.

### Просмотр контента

- Полноэкранная модалка просмотра изображения.
- Перелистывание фото внутри модалки кнопками.
- Перелистывание клавишами `ArrowLeft` и `ArrowRight`.
- Закрытие по `Esc`.
- Закрытие по клику на затемнение.
- Поддержка swipe-жеста для листания изображений.

### Работа с данными

- Предзагрузка следующей пачки изображений для более плавной бесконечной ленты.
- Предварительный прогрев кеша изображений, чтобы переходы были быстрее.
- Подтягивание данных о породе через API.
- Блок `Топ 5 ваших любимых пород` на странице избранного.

### Работа с избранным

- Логика избранного синхронизирована с `localStorage`.
- Во вкладке избранного можно открыть модалку просмотра так же, как и в общей ленте.
- Для карточек в избранном доступна кнопка `Поделиться`.
- При удалении текущего фото из избранного внутри модалки показывается следующее доступное изображение без поломки сценария.

### Шаринг

- Используется `Web Share API`, если браузер поддерживает нативный шаринг.
- Если нативный шаринг недоступен, ссылка копируется в буфер обмена.
- Если копирование недоступно, фото открывается в новой вкладке.
- Пользователь получает короткий toast-статус после действия.

## Логика блока «Топ 5 ваших любимых пород»

Рейтинг пород строится только по изображениям, которые пользователь добавил в избранное.

Правила сортировки:

1. Чем чаще одна и та же порода встречается в избранном, тем выше она в топе.
2. Если количество совпадает, выше поднимается порода, которая появилась в избранном позже.
3. В блок выводится не более 5 пород.
4. Если у изображения нет данных о породе, оно не влияет на рейтинг.

## API

Для загрузки изображений используется `TheCatAPI`.

Основной endpoint:

```text
GET /v1/images/search
```

В проекте используются параметры:

- `limit`
- `mime_types=jpg,png`
- `size=med`
- `has_breeds=true`

Благодаря `has_breeds=true` из API можно получать не только URL изображения, но и дополнительные данные о породе.

Примеры данных, которые может вернуть API вместе с фото:

- `breeds[].name`
- `breeds[].origin`
- `breeds[].temperament`
- `breeds[].description`
- `breeds[].life_span`
- `breeds[].alt_names`
- `breeds[].wikipedia_url`
- `breeds[].weight.metric`
- `breeds[].weight.imperial`
- `categories[]`
- `width`
- `height`

## Архитектура проекта

```text
src/
  api/          работа с TheCatAPI
  components/   UI-компоненты
  hooks/        пользовательские хуки с бизнес-логикой
  utils/        вспомогательные утилиты
  App.tsx       композиция приложения
```

Основные модули:

- `src/api/catsApi.ts` - запросы к API, нормализация данных, prefetch изображений.
- `src/hooks/useCatFeed.ts` - управление общей лентой, infinite scroll, prefetch следующей пачки.
- `src/hooks/useFavorites.ts` - логика избранного и синхронизация с `localStorage`.
- `src/components/CatViewerModal.tsx` - просмотр фото в модалке.
- `src/components/FavoriteBreedsSummary.tsx` - блок топа любимых пород.

## Локальный запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Создание `.env`

```bash
cp .env.example .env
```

Пример содержимого:

```bash
VITE_THE_CAT_API_KEY=your_api_key_here
```

### 3. Запуск dev-сервера

```bash
npm run dev
```

### 4. Production-сборка

```bash
npm run build
```

## Проверка качества

Для проекта использовались такие проверки:

```bash
npm run lint
npm run build
```

Обе команды должны выполняться без ошибок.
